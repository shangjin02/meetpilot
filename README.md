# MeetPilot AI｜智能会议助手

MeetPilot AI 是一个面向办公与学生项目团队的轻量 AI 会议助手 MVP。它不做视频会议本身，而是聚焦会议前后的高频痛点：会前准备、会后纪要、行动项提取与跟进。

## 产品介绍

产品流程：

```text
Create meeting
→ Generate pre-meeting preparation
→ Paste meeting notes or transcript
→ Generate meeting summary
→ Extract action items
→ Track follow-up tasks
```

核心标语：

```text
让每一次会议都有准备、有结论、有跟进
```

## 核心用户痛点

- 开会前不知道应该准备哪些资料和问题。
- 会后整理纪要耗时，结论和待办容易散落在聊天、文档和个人笔记中。
- 会议中提到的行动项缺少统一追踪，容易遗漏。
- 项目团队需要一个简单、可演示、可理解的会议工作流工具。

## 主要功能

- 工作台：查看今日会议、待准备会议、待整理纪要和待跟进事项。
- 我的会议：查看所有会议卡片，并进入会议工作区。
- 新建会议：创建会议并保存到 localStorage。
- 会前准备：模拟 AI 生成会议目标总结、议程、关键问题、准备清单、风险提醒和参会人关注点。
- 会议纪要：粘贴或载入示例会议文本，模拟 AI 生成摘要、结论、决策、未解决问题和跟进消息。
- 待办事项：从会议纪要中自动提取行动项，并同步到会议详情和全局待办页。
- 会议模板：提供周会、复盘、产品评审、客户沟通和实验分析会模板。
- 重置演示数据：一键恢复默认演示数据，便于考试或路演重复演示。

## 技术栈

- React
- TypeScript
- Vite
- Tailwind CSS
- lucide-react
- localStorage

本阶段不包含后端、登录、真实 LLM API、摄像头、麦克风、WebRTC 或录制能力。

## 本地运行

```bash
npm install
npm run dev
```

构建检查：

```bash
npm run build
```

## 推荐演示流程

1. 打开应用，展示“工作台”的会议和待办概览。
2. 进入“我的会议”。
3. 打开“产品增长复盘会”。
4. 点击“生成 AI 会前准备”，展示议程、关键问题、准备清单和风险提醒。
5. 进入“会议纪要”。
6. 点击“载入示例会议文本”。
7. 点击“生成 AI 会议纪要”。
8. 展示会议摘要、关键结论、决策事项、未解决问题、待办事项和跟进消息草稿。
9. 进入“待办事项”标签，展示自动提取的行动项。
10. 切换到全局“待办事项”页面，更新任务状态并筛选待办。

## 当前限制

- AI 内容由本地 mock 函数生成，不调用真实模型。
- 数据仅存储在浏览器 localStorage 中，不支持多设备同步。
- 不支持团队协作、权限管理和评论。
- 不支持真实日历接入、语音转文字或文档导出。
- 待办提取是规则化模拟，适合产品逻辑演示，不代表真实 NLP 能力。
